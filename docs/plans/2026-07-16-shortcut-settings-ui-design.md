# 快捷键设置 UI 设计

## 目标

在现有 Avalonia 设置区域提供快捷键下拉选择，避免用户直接编辑 `config.json`。默认仍为 Caps Lock，支持项目当前平台层已实现的键名。

## 交互与数据流

启动时从 `AppHost.Config.Shortcut` 初始化设置页的选中项。下拉框显示友好名称，内部值保存为配置键名，例如“Caps Lock”对应 `caps_lock`，“鼠标前进侧键”对应 `x2`。用户点击保存后，ViewModel 校验选项，将配置序列化写回原配置文件；保存成功后显示“已保存，重启后生效”。快捷键监听器不在运行中重建，避免释放和创建全局 Hook 时产生录音状态问题。

## 范围

第一版只编辑主触发键，保留当前 `keys` 多键配置的兼容读取；保存单选结果时写入 `shortcut.keys` 单元素数组，并同步 `shortcut.key`，确保旧逻辑和新逻辑都能读取。暂不提供组合键录制、模型路径、输出模式等设置。

## 错误处理与测试

配置文件写入采用临时文件替换或等价的安全写入方式；序列化失败、权限不足时保留原文件并显示错误。测试覆盖选项映射、保存后 JSON 字段、非法选项拒绝以及默认 Caps Lock。完成后运行核心测试和应用构建验证。
