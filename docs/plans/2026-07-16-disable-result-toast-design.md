# 取消识别成功 Toast

## 目标

语音识别成功后不再弹出系统 Toast，避免每次输入都产生打扰；识别、后处理、上屏、归档和错误提示保持不变。

## 方案

在 `DictationPipeline` 中移除识别成功后的通知触发路径，而不是只修改默认配置。这样即使用户已有旧版 `config.json`，也不会继续弹出成功通知。录音开始通知和错误通知仍由现有配置控制。

删除 `NotificationConfig.ShowOnResult` 及其热重载和文档字段。旧配置中的未知字段会被 JSON 反序列化忽略，保持兼容。

## 验证

新增流水线回归测试，模拟一次长按录音和成功识别，断言文本仍然产出且通知处理器调用次数为零；同时运行 Core 全量测试并构建 App 项目。
