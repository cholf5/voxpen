# 无标点 ASR 的标点模型下载设计

## 目标

为不具备原生标点能力的 ASR 模型提供应用内、可选的外挂标点模型下载。用户无需手动下载或解压 CT-Transformer 标点模型；未安装标点模型时，VoxPen 仍可正常工作，只输出无标点文本。

## 下载页与交互

设置页根据当前选中 ASR 的 `EngineCapabilities.Punctuation` 动态展示标点模型区域。原生带标点的模型不展示下载入口，改为提示“此模型自带标点”。无原生标点的模型展示标点模型说明、安装状态、独立的下载/取消按钮和进度。

标点模型下载仅负责安装文件到固定的 `models/Punct-CT-Transformer/...` 目录；不隐式修改配置或切换运行实例。下载完成后，用户通过既有“保存并应用”统一加载生效。ASR 和标点下载均支持续传、取消和独立失败处理。

## 架构与数据流

Core 将 ASR 模型定义与可下载资源解耦，新增固定的标点模型包定义：下载地址、包名、默认目录和校验规则。通用下载协调器接收该资源定义，复用已有 HTTP Range 续传、解压安装、取消和进度状态；Windows 平台下载实现不增加专用分支。

`AppHost` 新增标点模型下载入口。ViewModel 分别维护 ASR 和标点下载状态，避免相互覆盖。保存并应用时，候选宿主先加载 ASR；若引擎无原生标点且本地标点模型有效，再加载 `SherpaPunctuator`，否则使用 `NullPunctuator`。原生标点模型始终不加载外挂标点，避免重复处理。

## 状态、错误处理与验证

模型状态拆分为“识别模型”和“标点模型”。识别模型缺失仍为阻断性错误；标点模型缺失显示“未安装，将输出无标点”，但不阻止启动或保存。下载期间禁用设置编辑及两个下载入口，防止切换目标；取消后保留 `.partial` 文件以便续传。安装后校验 `model.onnx`，失败不覆盖已有有效模型。

测试覆盖标点资源定义、下载协调器、安装校验、ViewModel 可见性与状态映射，以及 `AppHost` 的原生标点、外挂标点成功和缺失/加载失败降级路径。README 更新为：无标点 ASR 可在设置页可选下载标点模型，不再要求手动解压。
